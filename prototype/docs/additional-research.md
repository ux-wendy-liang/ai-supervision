## 一、定价策略研究

### 1.1 行业定价模式对比

教练/咨询平台主要有两种收费模式：

| 模式 | 特点 | 适合场景 |
|------|------|---------|
| **按次收费 (Per-Session)** | $100-$250/次，灵活但单价高 | 低频用户、试用期 |
| **订阅制 (Subscription)** | $50-$135/周，包月更划算 | 高频用户、长期关系 |

### 1.2 竞品定价对比

#### 企业级平台（B2B）

| 平台 | 定价模式 | 价格区间 | 平台抽成 |
|------|---------|---------|---------|
| **BetterUp** | 定制报价 | $75-$250/次 | 不透明 |
| **CoachHub** | 定制报价 | $150-$300/次 | 30-40% |

> 企业级平台定价不透明，需要联系销售获取报价

#### 个人用户平台（B2C）

| 平台 | 定价模式 | 价格区间 | 平台抽成 |
|------|---------|---------|---------|
| **Clarity.fm** | 按分钟收费 | $2-$30/分钟 | **15%** |
| **BetterHelp** | 订阅制 | $65-$100/周 | N/A |
| **Talkspace** | 订阅制 | $69-$109/周 | N/A |
| **Open Path** | 按次收费 | $30-$70/次 + $65会员费 | N/A |
| **Noomii** | 年费上架 | $447/年 | 无抽成 |
| **Profi.io** | SaaS订阅 | $39/月起 | 无抽成 |

### 1.3 CoachSpace 定价建议

#### 对教练端（供给侧）

建议采用 **低抽成策略** 吸引早期教练：

```
推荐方案：
- 免费入驻
- 平台抽成 10-15%（低于 CoachHub 的 30-40%）
- 或 $29-49/月 订阅 + 0% 抽成（类似 Profi.io）
```

**核心卖点**：
- "保留 85%+ 收入"（对比 CoachHub 只能拿 60-70%）
- "无隐藏费用"
- "10 分钟完成注册"

#### 对客户端（需求侧）

| 定价方式 | 说明 |
|---------|------|
| **教练自主定价** | 让教练根据经验和认证设置价格 |
| **建议价格区间** | 提供行业基准，帮助新教练定价 |
| **首次优惠** | 首次咨询可打折，降低尝试门槛 |

## 二、教练 Profile 页面最佳实践

### 2.1 关键转化指标

| 指标 | 行业基准 | 优秀水平 |

| 首次印象时间 | 2.6 秒 | - |

### 2.2 高转化 Profile 页面核心要素

#### A. 视觉层级（First Impression）

```
优先级排序：
1. 专业头像（大尺寸、高清）
2. 姓名 + 核心头衔
3. 评分 + 评价数量

```

**关键数据**：
- 89% 的用户信任他人推荐
- 73% 的用户信任专家资质
- 71% 的用户信任"既专业又有同理心"的专家

#### B. 信任信号（Trust Signals）

| 信任元素 | 重要性 | CoachSpace 现状 |
|---------|--------|----------------|
| 认证徽章（ICF、PCC 等） | 高 | ✅ 已有 |
| 客户评价 | 高 | ✅ 已有 |
| 视频介绍 | 中高 | ❌ 缺失 |
| 成功案例数据 | 中 | ❌ 缺失 |
| 媒体背书 | 中 | ❌ 缺失 |

#### C. 核心内容模块

**现有（已实现）**：
- ✅ 专业领域标签
- ✅ 关于介绍
- ✅ 教练方法
- ✅ 认证资质
- ✅ 客户评价
- ✅ 可预约时间

**建议添加**：
- ❌ 视频自我介绍（30-60秒）
- ❌ 成功案例/客户故事
- ❌ 回复时间（如"通常 24 小时内回复"）
- ❌ "为什么选择我"模块

## 三、预约流程 UX 研究

### 3.1 预约流程优化的重要性

> "如果你的网站依赖预约来获客，每一次点击、滚动和表单字段都很重要。流程越顺畅，转化率越高。**优化预约流程可提升 20-40% 的预约量**。"

### 3.2 预约流程关键步骤

```
理想预约流程（4步）：
1. 选择服务类型 → 清晰的选项说明
2. 选择日期时间 → 智能默认、高亮推荐
3. 填写基本信息 → 最少字段、行内验证
4. 确认并支付 → 透明价格、安全感
```

### 3.3 各步骤优化要点

#### Step 1: 服务选择

| 问题 | 解决方案 |
|------|---------|
| 选项太多让用户困惑 | 添加"推荐"或"最受欢迎"标签 |
| 不知道哪个适合自己 | 添加简短描述或"适合谁" |

#### Step 2: 时间选择

| 问题 | 解决方案 | 效果 |
|------|---------|------|
| 日历组件难用 | 移动端友好的日期选择器 | 某公司修复后转化提升 34% |
| 不知道选哪个时间 | 高亮"推荐时段"或"热门时段" | 减少决策疲劳 |
| 时区困惑 | 自动显示用户本地时区 | 减少预约错误 |

#### Step 3: 信息收集

| 最佳实践 | 说明 |
|---------|------|
| **减少字段** | 从 6 个减到 3 个，转化提升 66% |
| **行内验证** | 实时显示错误，不要等提交 |
| **允许修改** | 填错后可以改，不要重新开始 |
| **智能填充** | 支持浏览器自动填充 |

#### Step 4: 确认与支付

| 最佳实践 | 说明 |
|---------|------|
| **价格透明** | 提前显示总价，避免"惊喜费用" |
| **添加价格明细** | "可展开的价格明细"减少支付前流失 |
| **取消政策** | 显示"24小时内可免费取消" |
| **安全标识** | 显示支付安全徽章 |

### 3.4 移动端特别注意

移动端是大多数用户的使用场景，但也是预约流程最容易出问题的地方：

| 问题 | 影响 |
|------|------|
| 日期选择器在小屏幕不好用 | 某公司因此流失 60% 移动用户 |
| 表单字段太小 | 输入困难，放弃预约 |
| 页面加载慢 | 超过 3 秒就会流失用户 |

**优化建议**：
- 使用原生日期选择器
- 增大点击区域（最小 44x44px）
- 页面加载控制在 1 秒内

### 3.5 预约后的体验

| 功能 | 重要性 |
|------|--------|
| 即时确认邮件 | 高 |
| 日历邀请（.ics 文件） | 高 |
| 短信提醒 | 中（减少 no-show） |
| 轻松改期/取消 | 中 |

## 四、信任与安全机制研究

### 4.1 为什么信任很重要

> **76% 的消费者**表示，对品牌的信任是做出购买决定时最关键的因素。

对于双边市场（如 CoachSpace），信任是核心：
- 客户需要信任教练的专业性
- 教练需要信任平台的支付和客户质量

### 4.2 信任建立的核心要素

#### A. 身份验证（Identity Verification）

| 验证级别 | 方法 | 信任度 |
|---------|------|--------|
| 基础 | 邮箱验证 | 低 |
| 中等 | 手机 + 邮箱 + LinkedIn | 中 |
| 高级 | 政府ID + 资质证书 | 高 |

**对 CoachSpace 的建议**：
- 教练端：要求 LinkedIn 连接 + 资质证书上传
- 客户端：邮箱 + 手机验证即可

#### B. 验证徽章（Verification Badges）

常见的信任徽章类型：

| 徽章类型 | 示例 |
|---------|------|
| 身份验证 | ✓ Verified Identity |
| 资质认证 | 🏅 ICF Certified |
| 背景调查 | ✓ Background Checked |
| 平台推荐 | ⭐ Top Rated |
| 活跃状态 | 🟢 Usually responds within 24h |

#### C. 评价系统（双向评价）

| 功能 | 说明 |
|------|------|
| 客户评价教练 | 常见，已实现 |
| 教练评价客户 | **保护教练**，避免问题客户 |
| 评价验证 | 只允许已预约用户评价 |
| 评价回复 | 让教练可以回应评价 |

#### D. 支付保障

| 功能 | 说明 |
|------|------|
| 第三方支付 | Stripe、PayPal 等安全标识 |
| 资金托管 | 服务完成后才放款给教练 |
| 退款政策 | 清晰的取消和退款规则 |
| 争议处理 | 透明的争议解决流程 |

### 4.3 信任信号的展示位置

```
Profile 页面：
├── 头像旁 → 验证徽章
├── 姓名下 → 认证标签（ICF、PCC 等）
├── 侧边栏 → 回复时间、接单数量
└── 底部 → 安全保障说明

预约确认页：
├── 教练信息 → 简要资质展示
├── 支付区域 → 安全支付徽章
└── 底部 → 取消政策、客服联系方式
```

### 4.4 CoachSpace 信任机制现状 vs 建议

| 功能 | 现状 | 建议 |
|------|------|------|
| 认证徽章 | ✅ ICF 等认证 | 添加 Verified 徽章 |
| 客户评价 | ✅ 已有 | 添加评价验证标识 |
| 双向评价 | ❌ 无 | 考虑添加 |
| 回复时间 | ❌ 无 | 添加"通常 X 小时内回复" |
| 视频介绍 | ❌ 无 | 建议添加 |
| 支付安全 | ❌ 无展示 | 添加安全徽章 |
| 取消政策 | ✅ 有 | 可更醒目 |

## 五、设计师可执行的研究清单

### 5.1 竞品体验走查

| 活动 | 怎么做 | 产出 |

| **预约流程** | 在 BetterHelp 或 Talkspace 完成预约流程 | 流程图 + 体验报告 |
| **Profile 页面** | 截图 10+ 个教练 Profile | 设计模式总结 |
| **移动端体验** | 用手机完整走一遍流程 | 移动端问题清单 |

### 5.2 用户访谈问题

**对潜在客户（找教练的人）**：
1. 你选教练时最看重什么？（排序）
2. 什么会让你放弃预约？
3. 你会在预约前想了解教练什么信息？
4. 你对"AI 辅助教练"怎么看？

**对教练（供给端）**：
1. 你现在怎么获取客户？
2. 你选择入驻平台时最看重什么？
3. 你能接受多少比例的平台抽成？
4. 你希望平台提供什么工具？

### 5.3 可用性测试任务

```
任务 1：找教练
"假设你想找一个帮助你职业转型的教练，请在这个网站上找到一个合适的教练。"
- 观察：搜索行为、筛选使用、页面浏览路径

任务 2：预约流程
"请预约这位教练的一次咨询。"
- 观察：选择时间的困难、表单填写问题、支付犹豫

任务 3：了解教练
"请告诉我这位教练是否值得信任，你从哪些信息判断的？"
- 观察：用户关注哪些信任信号
```
